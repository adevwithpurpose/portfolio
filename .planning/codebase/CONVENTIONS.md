# Conventions

Source: README, AGENTS, docs/README.

## Public positioning

Keep broad but premium:
- automation systems
- web development/web systems
- AI assistants/internal tools
- workflow cleanup
- operational visibility

Avoid making the main portfolio about Shopify, n8n, or generic e-commerce unless Safeer explicitly asks for a niche landing page/campaign.

## Pricing posture

Use package/value framing, not cheap hourly framing. Current README examples:
- Website / landing page build — from $750
- Automation system build — from $500
- AI assistant / internal tool — from $900

Avoid public `$18/hr`, `$20/hr`, `$25/hr` positioning.

## Coding rules

- Surgical changes only.
- Do not refactor unrelated files.
- For Next.js changes, read relevant local Next docs first.
- Verify with `npm run lint` and `npm run build` before deploy claims.

## Docs rules

- Files prefixed `legacy-` are source/historical material, not current direction.
- Prefer additive updates over restructuring docs.
- Keep AGENTS lean; route detailed workflows to docs, skills, or this map.
