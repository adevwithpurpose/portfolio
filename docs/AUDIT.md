# Portfolio Docs Audit

Audited: 2026-04-26

## Summary

The folder had strong raw material but was messy because it mixed:

- strategy docs
- market/channel research
- outreach templates
- niche-specific Shopify/n8n docs
- LinkedIn profile assets
- progress tracking

The new structure separates these into numbered folders and marks niche-specific content as `legacy-` where it should not drive the main generic portfolio positioning.

## Key decisions

### 1. Main public portfolio stays generic

Use: automation, web development, workflow systems, AI assistants/internal tools.

Avoid as homepage positioning: Shopify, n8n, e-commerce-only, tool-list identity.

### 2. Niche docs are preserved, not deleted

Shopify/n8n/e-commerce docs are useful for:

- targeted landing pages
- proposals
- examples
- outbound campaigns

But they should not define the main portfolio homepage.

### 3. Current strongest proof asset

`05-linkedin-presence/assets/flagship-proof-asset-public-posting-version.md`

This should become the first public proof/thought-leadership page. It is generic enough if we remove founder-led specificity where needed.

### 4. Current weakest area

The docs have many plans and templates but not enough verified public proof:

- real project screenshots
- before/after metrics
- deployed links
- anonymized case studies
- testimonials

## Vague / risky items found

1. Pricing claims and package sizes need validation against Safeer's actual proof and target client budgets.
2. Upwork/channel volume claims need current verification before using publicly.
3. LinkedIn automation tooling claims need verification because platform rules and automation risk change often.
4. Cold email templates need deliverability compliance review before use at scale.
5. “AI agent retainer” language may be too narrow/hype-heavy for generic portfolio homepage.
6. Some docs still assume Shopify/n8n positioning and must be generalized before reuse.
